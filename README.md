## IP被封，先搞清楚是哪种封

不是所有的"连不上"都是IP被封，也不是所有的IP被封都一样处理。

**三种常见情况，症状不同：**

1. **TCP阻断**：SSH/HTTP全部超时，但ICMP（ping）有时还通。这是最常见的GFW针对性封锁，通常发生在流量特征被识别之后。
2. **ICMP阻断**：ping不通，但某些端口的TCP连接还能建立。相对少见，有时是运营商级别的过滤。
3. **完全封锁**：ping不通、TCP全超时、UDP也没响应。这种基本宣告这个IP在国内方向彻底废了。

判断方法很简单：用国内的在线ping工具（比如站长工具的多地ping）测你的IP，同时从境外节点测。境外能通、国内全超时——TCP阻断无疑。境外境内都不通——要么机器宕机，要么完全封锁。

有一点要注意：换IP之前，先确认你的服务配置（反代、域名解析、防火墙规则）是否绑定了旧IP，换完要同步更新，不然换了也白换。

**方案二：用DMIT自带的流量中转或Proxy功能**

部分套餐支持流量中转，即通过DMIT的中转节点绕过被封的IP直接出口。这个功能不是所有套餐都有，具体要看你购买的产品说明。

**方案三：换节点或升级线路**

如果你的IP反复被封，换IP只是治标。根本原因可能是：

- 当前线路的IP段整体质量差，换一个还是同段的IP
- 使用方式触发了流量特征识别（比如高频小包、特定端口）
- 所在地区的运营商对该线路有针对性过滤

这种情况下，升级到Premium线路或者换一个地区的节点，比反复换IP更有效。

## 怎么降低IP被封的概率

换IP是被动应对，主动预防才是正解。

**端口选择很关键。** 22、80、443这几个端口是GFW重点审查对象，尤其是22端口的SSH流量特征非常明显。把SSH改到高位端口（比如5000以上的随机端口），被识别的概率会下降不少。

**流量混淆。** 如果你在VPS上跑代理服务，协议选择和混淆方式直接影响存活周期。裸SOCKS5基本是送人头，带TLS的协议配合CDN中转，存活时间差距可以是十倍以上。

**不要在高峰期跑大流量。** 晚上8点到11点是国内网络审查最活跃的时段，这个时间段跑大量出境流量，被命中的概率显著更高。

**IP存活周期监控。** 可以用简单的cron脚本定期从国内探针测试IP连通性，一旦发现阻断立即收到通知，比等到用的时候才发现要好处理得多。

## FAQ

### DMIT换IP要多少钱，流程复杂吗？

换IP的费用因节点不同而有差异，具体金额在后台提交工单时会显示。流程不复杂：登录后台，找到对应VPS，选择IP更换选项，付款后等待处理。一般几小时内完成，不需要重装系统，数据不受影响。

### 香港节点和洛杉矶节点，哪个更不容易被封？

这个问题没有绝对答案，取决于你的使用场景和所在地区的运营商。香港Premium延迟更低，但香港IP段整体上被审查的频率也更高，因为用的人多。洛杉矶CN2 GIA的IP段相对"冷门"一些，被批量封的情况少，但延迟比香港高30-50ms左右。如果你在意延迟，选香港；如果你更在意稳定性，洛杉矶Pro是更保险的选择。

### IP被封了，数据还在吗？

在的。IP封锁只影响网络层面的可达性，VPS本身的数据、系统、配置全部完好。换IP之后，SSH用新IP连进去，一切照旧。

### DMIT支持退款吗？

DMIT有退款政策，新购订单在一定时间内可以申请退款，具体条款以官网当前政策为准。如果是因为IP被封导致无法使用，建议先提工单说明情况，客服有时候会给出换IP或补偿的方案，不一定要走退款流程。

### Eyeball线路和Pro线路，IP被封的频率差多少？

我自己的体感是差距明显。Eyeball线路的IP段质量参差不齐，有些IP买来就是"二手"的，可能已经被封过一次又解封了，再次被封的概率很高。Pro线路的IP段相对干净，加上线路本身的流量特征更接近商业流量，存活周期通常更长。如果你对稳定性有要求，Eyeball线路省下来的那点钱，很可能会花在反复换IP上。

### 工单响应速度怎么样？

工作日响应比较快，通常几小时内有回复。周末和节假日会慢一些，有时候要等到下一个工作日。紧急情况建议工单里注明"IP blocked, urgent"，优先级会高一些。
